Abstract

Monster Party is designed by Andreea, George and Robert. It started out as a school project but it looks like it will have a bright future. We are a group of computer science students, trying to learn as much as possible about web technologies in a shortest amount of time, developing an awesome app as a side-product of this process. You will hear from us again for sure.

Introduction

The story centers on our two angry monsters, Roby and Juan. They try to escape a complicated puzzle world in order to have a peaceful life.

The high-level goals of the game are to:

  • Improve Problem Solving Skills.
  • Improve Concentration! Concentration is required to solve a brainteaser. Therefore puzzles can help increase the attention span in both adults and children.
  • Improve Cognitive Ability
  • Improve Productivity.

Structure

We tried to take things step by step while still being able to view progress as we move on. For this part of the assignemt we took care a little bit about the design of our application because functionality isn’t everything. Like any web application our application have the following components:

  • Static contents like HTML.
  • Client side files like CSS and Javascript
  • Externel resourses like images and fonts

Web Application Root Directory

This is the main or Root folder of web application. Usually name of this folder becomes your web application context. http://localhost:81/CLIW/WebPuzzle/

Authenticated directory

This is the special directory under web application root directory. This is special because the files available within this folder will not be accessible to client directly, he will have to login to have access to them.

Anauthenticated directory

This directory contains files that can be accessed by any client.

Style directory

This directory containts all the CSS file which were used in the project.

Resources directory

This directory containts all the images and fonts which were used in project.
